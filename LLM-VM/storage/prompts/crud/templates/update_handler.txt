Inputs

1. A user query describing what to update/modify.

====================
Your Tasks
====================
1. Identify which candidates match the user's update intent.
2. Generate updated content for each selected node.
3. Preserve unchanged fields exactly as they are.
====================
Selection Criteria (CRITICAL)
====================
**Be conservative**: Update ONLY nodes that clearly match user intent.
- When uncertain, prefer NOT updating over making incorrect changes.
- If the query is vague and multiple candidates could apply equally, update none.

**Using Semantic Scores**:
- Score >= 0.6: Likely match - proceed if context supports
- Score 0.4-0.6: Medium confidence - only update if explicitly named or clearly referenced
- Score < 0.4: Low confidence - do NOT update unless the query explicitly names this node

**When to Return Empty Updates**:
- No candidates semantically match the entity mentioned in the query
- Query references a node that doesn't exist in candidates
- Query is ambiguous and could apply to multiple candidates equally
- Semantic scores are too low to confidently identify the target

====================
Field Preservation Rules
====================
- Include ALL original fields in the output (even unchanged ones)
- Only modify fields explicitly mentioned or implied by the request
- Maintain field types: lists stay lists, strings stay strings
- For partial updates (e.g., time change), preserve description, highlights, etc. exactly
====================
Output Format
====================
Output a single JSON object:
```json
{{
  "updates": [
    {{
      "id": "1",
      "path": "Day 1 > Node Name",
      "fields": {{ ... }},
      "new_type": null,
      "reasoning": "Changed X from Y to Z"
    }}
  ],
  "reasoning": "Overall explanation"
}}
```

- `id`: The candidate number (e.g., "1" for candidate [1])
- `path`: The node's tree path
- `fields`: Complete field set including unchanged fields
- `new_type`: Only if type changes (e.g., "Restaurant"), otherwise null
- `reasoning`: Brief explanation of what changed and why

**Empty response** when no clear match:
```json
{{
  "updates": [],
  "reasoning": "No candidates clearly match because [specific reason]"
}}
```
