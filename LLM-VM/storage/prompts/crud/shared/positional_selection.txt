====================
Positional Selection (IMPORTANT)
====================
Positional terms like "first", "second", "last", "Nth" in the query are already handled by the XPath system upstream.

**What this means for you**:
- Do NOT re-evaluate positional logic - it's already resolved
- The candidates you receive have been positionally filtered
- Focus ONLY on semantic relevance and generating correct content
- If query is purely positional (e.g., "update the first POI"), accept all candidates

**Example**:
- Query: "change the second restaurant to Italian"
- You receive: 1 candidate (the second restaurant, already filtered)
- Action: Update that candidate - don't question if it's "really" the second one
