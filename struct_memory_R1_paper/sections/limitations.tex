\section{Limitations}
The present formulation of \textsc{Struct Memory-R1} has several important limitations.
\textbf{(1) Oracle structure.} The current results rely on oracle human-judged structure for memory construction. While the Memory Manager is designed for end-to-end training, its RL loop has not yet been completed; the reported gains therefore represent a ceiling on what tree structure can provide.
\textbf{(2) Fixed schema.} The Memory Manager performs CRUD updates under a stable induced schema but does not yet support general tree restructuring (e.g., splitting or merging branches) during training.
\textbf{(3) Single-pass retrieval.} The Retrieve Agent emits a single search plan rather than acting interactively one level at a time with intermediate observations. Multi-turn tree traversal could improve recall on complex queries.
\textbf{(4) No hybrid retrieval.} The method emphasizes explicit tree search rather than dense vector retrieval, so it does not yet address the hybrid regime in which semantic similarity and structural routing are both learned.
\textbf{(5) Single benchmark.} Evaluation is limited to LoCoMo; broader validation on additional long-horizon conversational benchmarks is needed.
